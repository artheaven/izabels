
# SEO ТЗ уровня «БОГ» для цветочного и подарочного магазина в Болгарии (2025)
**Архитектура: Next.js 15 (SSR/SSG/ISR) на Vercel + Node/Prisma API на Railway. Мультиязычность: bg (дефолт), en, ru. Локальный рынок: Болгария.**  
**Цель:** доминирование в органике и локальном паке (Maps) по кластерам «доставка цветов», «букеты», «подарки», «праздники/именни дни» в городах (Варна/София/Пловдив/Бургас) + рост транзакций и LTV.

---

## 0) KPI и контроль качества
- **Квартальные KPI:**  
  - 80% ключевых страниц в топ‑10 по целевым кластерам (Varna + «всеболгарские» запросы).  
  - **Core Web Vitals (мобайл):** LCP ≤ 2.5s, **INP ≤ 200 ms**, CLS ≤ 0.1. citeturn11search2turn11search4turn11search7  
  - **GBP (Google Business Profile):** рейтинг ≥ 4.7, ≥ 20 свежих отзывов/квартал, ответ ≤ 24ч; отсутствие предупреждений о фейковых отзывах. citeturn10news20  
  - **Merchant/Рич‑сниппеты:** 90% карточек валидны в Rich Results Test; Product/Offer/Shipping/ReturnPolicy на месте. citeturn3search1turn3search7turn3search0

- **Мониторинг:** GA4 + GSC + CrUX + Lighthouse CI на PR. Документация Page Experience/INP — ориентир. citeturn11search2

## 2) Структура URL и мультиязычность
- Главный домен: `example.bg` (ccTLD усиливает геосигнал) **или** `.com` с жёстким hreflang и локализацией. ccTLD — геосигнал, но не прямой фактор ранжирования. Геотаргетинг через язык/контент/hreflang; отчёт International Targeting в GSC **депрекейтнут**. citeturn5search8turn5search15turn5search16
- Локали:  
  - BG (дефолт): `/`  
  - EN: `/en/`  
  - RU: `/ru/`
- **hreflang** на все версии, каноникал между вариантами не ставим (разные языки ≠ дубликаты).

- **Слаги**: на BG допустимы кириллические, но для стабильности ссылок/аналитики рекомендуем латиницу (транслит).

## 3) Техническое SEO (Next.js 15 + Vercel)
1. **Рендеринг:** SSG/ISR для статических лендингов (город/праздник), SSR — для каталога/фильтров. **Мобайл‑перф:** добиваем INP ≤ 200 ms. citeturn11search2  
2. **Metadata API**: шаблоны `<title>`, `<meta name="description">`, OG/Twitter, canonical/robots на уровне layout. citeturn7search4  
3. **robots.txt**: запрещаем параметры пагинации/фасетов, если есть каноникал на чистую категорию.  
4. **Sitemap‑ы:** корневой `sitemap.xml` + индекс языков и типовые карты: `/sitemap-pages.xml`, `/sitemap-products.xml`, `/sitemap-images.xml`. Hreflang внутри карт.  
5. **JavaScript SEO:** избегать критичного контента, появляющегося только после клиентского JS; важные блоки — SSR. Руководство по JS SEO — обязательные проверки. citeturn7search0  
6. **Ленивая загрузка:** native `loading="lazy"` + **фиксированные width/height** для снижения CLS; убедиться, что Google видит lazy‑контент. citeturn6search0turn6search3  
7. **Картинки:** WebP/AVIF, **стабильные URL** для повторного использования/кэша; alt на BG/EN/RU. citeturn6search1turn6search4  
8. **Перелинковка и хлебные крошки:** навигационные breadcrumbs + JSON‑LD `BreadcrumbList`. (Даже если на мобильных Google визуально урезает элементы, микроразметка остаётся полезной.) citeturn9search0  
9. **Site name:** `WebSite` schema для контроля отображаемого имени сайта; также `og:site_name`. citeturn9search1
10. **Core Web Vitals:** CI‑гейт по LCP/INP/CLS; бюджет запросов к API/шрифтам; preconnect к платежному провайдеру/CDN. Цели — см. §0. citeturn11search2

## 4) Структурированные данные (JSON‑LD)
> **Важно:** Разметка не является фактором ранжирования сама по себе, но расширяет сниппеты и Merchant‑опыт. citeturn2search7

Минимальный набор:
- **Organization** + `LocalBusiness` (Florist) на главной/контактах: NAP, геокоординаты, часы, `sameAs` соцсетей.  
- **Product/Offer** на PDP: цена BGN, доступность, SKU/GTIN (если есть), фото, `aggregateRating` (только свои 1st‑party отзывы). citeturn3search1  
- **ShippingService** (политика доставки) + `MerchantReturnPolicy` (уровень организации) — показы доставки/возвратов в сниппетах/панелях. Указать **applicableCountry (BG)**. citeturn3search7turn3search4turn3search10

Пример (укорочено):
```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Букет рози “Classico”",
  "image": ["https://example.bg/img/rose-classico-1.webp"],
  "sku": "BKT-000123",
  "offers": {
    "@type": "Offer",
    "priceCurrency": "BGN",
    "price": "89.00",
    "availability": "http://schema.org/InStock",
    "shippingDetails": {
      "@type": "OfferShippingDetails",
      "shippingRate": {
        "@type": "MonetaryAmount",
        "value": "6.90",
        "currency": "BGN"
      },
      "deliveryTime": {
        "@type": "ShippingDeliveryTime",
        "handlingTime": {"@type":"QuantitativeValue","minValue":0,"maxValue":1,"unitCode":"HUR"},
        "transitTime": {"@type":"QuantitativeValue","minValue":1,"maxValue":24,"unitCode":"HUR"}
      }
    }
  }
}
```
И на уровне организации (один раз на сайте):
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Izabel Flower",
  "url": "https://example.bg",
  "hasMerchantReturnPolicy": {
    "@type": "MerchantReturnPolicy",
    "applicableCountry": "BG",
    "returnPolicyCategory": "https://schema.org/MerchantReturnFiniteReturnWindow",
    "merchantReturnDays": 14,
    "returnMethod": "https://schema.org/ReturnByMail",
    "returnFees": "https://schema.org/FreeReturn"
  }
}
```

## 5) Шаблоны страниц (мета/контент/внутренняя перелинковка)
### 5.1 Главная
- **Hero‑слайдер** (акции/поводы месяца), блоки «город + доставка за X часа», «популярные букеты», «подарки к букетам».
- **SEO‑блок‑текст** 300–500 слов (BG) снизу: уникальный, «почему у нас», зоны доставки, часы, способы оплаты.
- **Разметка:** `Organization`/`LocalBusiness`, `BreadcrumbList`, избранные `Product`.

**Title:** `Доставка на цветя във Варна — Букети за Цветница, 8 март, Св. Валентин | Brand`  
**Desc:** `Поръчайте букет с доставка в рамките на 2–3 часа. Рози, лалета, кутии с цветя и подаръци. Плащане онлайн.`

### 5.2 Каталог + фильтры
- Боковая навигация по категориям, фасеты по **цена**, **размер S–XL**, **повод**, **цвет** и т.д.  
- **SEO‑требования к фасетам:** индексируем только **коммерчески ценные** пресеты (ручной allowlist); остальные — `canonical` на базовую категорию, `noindex,follow` для мусорных комбинаций.  
- **Сортировки**: по цене, названию, «сначала со скидкой».

### 5.3 PDP (страница товара)
- Крупное фото + 10 миниатюр, зум/лайтбокс; выбор размера/цвета упаковки/открытки; артикул генерируется автоматически.  
- Блок «С этим берут» (cross‑sell), «Доставка и връщане» (видимые правила).  
- **Разметка:** `Product/Offer` + Shipping/Returns.  
- **Варианты:** одна каноническая URL, варианты — через `offers`/`additionalProperty`.

### 5.4 Лендинги городов/районов/поводов
- Уникальный интро‑текст 300–600 слов, блок «время доставки/стоимость по району», топ‑наборы букетов + подарки.
- Внутренние ссылки: город → поводы → категории → PDP.

### 5.5 Блог/гиды
- Длинные руководства с таблицами и визуалами, **оригинальные фото** (студийные, геоконтекст — Болгария).  
- Секции «Краткий ответ» (для AI Overviews/Featured snippets): 1–2 абзаца + маркированный список; FAQ на странице. (FAQ‑rich‑results урезаны, но для пользователей/AI‑обзоров полезно). citeturn2search8

## 6) Локальное SEO (Google Business Profile)
- **Полная карточка**: категория *Florist* + *Flower delivery*, адрес/часы/телефон, сервис‑ареа, услуги, цены «от…», **товары** (букеты) с UTM на PDP, посты‑акции 1–2/нед.  
- **Медиа:** регулярные фото‑сессии (витрина, процесс, доставка).  
- **Отзывы:** автоматизация запросов после заказа; **никаких стимулов** (скидки за отзыв запрещены). citeturn10news20  
- **Q&A:** заранее заполнить типовые вопросы (доставка, сроки, способы оплаты).

- **Двойное ценовое отображение BGN/EUR:** подготовиться под обязательства в 2025–2026 (юзабилити + доверие). citeturn8search9turn8search19

## 10) Фасетная навигация и индексация
- Индексируем **только ценные** комбинации (например, `/bouquets/roses?occasion=tsvetnitsa`).  
- Все прочие комбинации: `canonical` на чистую категорию + `noindex,follow`; ссылки пагинации — `rel="next/prev"` (внутренняя логика, даже если Google не использует сигнал).  
- Убедиться, что фильтры не создают миллионы «тонких» страниц.

## 11) UX/скорость (влияет на ранжирование и конверсию)
- **Шрифты:** `display: swap`, preconnect к CDN, subset кириллицы.  
- **Кэширование:** ISR с revalidate для категорий/городов в пиковые сезоны (Цветница, 8 март, 15 септември).  
- **Оптимизация изображений:** Next/Image, `sizes`/`srcset`, AVIF/WebP, без лишних ретранскодирований; **стабильные URL**. citeturn6search4

## 12) Правила ревью и сниппетов
- Встраиваем **собственные отзывы** покупателей на PDP (про товары/доставку); **не агрегировать** чужие рейтинги — риск санкций и снятия сниппетов. FAQ‑rich‑results в 2023 сильно урезаны; считаем их бонусом, а не стратегией. citeturn2search8turn2search12  
- Ориентир по качеству отзывов/обзоров — рекомендации Google Reviews System. citeturn10search0

## 13) Платежи и SEO

## 14) Логирование, алерты, регресс‑тесты
- **CI‑проверки:**  
  - Lighthouse (мобайл) ≥ 90; Web Vitals гейт — падение → фейл билда.  
  - Валидатор схем (Rich Results Test API) — критичные ошибки = блок мержа.

- **Алерты:** просадка кликов/импрессий в GSC по ключевым кластерам; рост 404; ухудшение INP/LCP.
